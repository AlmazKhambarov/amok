What editors does it support?

Amok is a command line tool, so it is editor agnostic and works regardless of what editor you choose.

There are plans for integration with various editors for a better editing experience, but this is pretty far in the future

It should integrate easily into the build settings of your favorite editor however

What frameworks does it support?

Amok is framework agnostic, if it runs on the client and does not get collected it can be re-compiled.

What does it mean if a script has been collected?

A script can be collected by the virtual machine if it is a top to bottom, linearly executing script without any callbacks, for example, lets say we have a script called hello.js that contains the following source code. 

console.log('hello world');

This sort of script would run from top to bottom, and because there are no callbacks or other code that is recurring, it gets collected by the runtime.

How is this different from live-reload?

Amok does re-compilation of source code through the runtime, where-as live reload just does a simple page *reload* which is the equivalent of hitting the refresh button in your web browser.

Doing re-compilation lets us edit our algorithms, event handlers and other types of closures while the application is running without having it restart no loosing state.
