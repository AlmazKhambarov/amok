What Is It?

Amok is a free, cross-platform, open source standalone command line tool for rapid prototyping, development and debugging of JavaScript applications.

Amok is written in JavaScript, runs on node and is distributed via npm

 

What Does It Do?

Amok monitors changes in the file system. As soon as you save a file, it is then preprocessed, compiled and bundled as needed, and reloaded in the browser session without refreshing. This keeps the application state unchanged while making edits to the source code.

Additional features include a zero configuration http server and an interactive mode (read-eval-print-loop).

What Compiler Can It Use?

Amok can use any compiler that has a command line interface capable of writing to a file.

This is done trough a simple command line option or environment variable configuration, which means it can work with CoffeeScript, TypeScript, Babel, Webpack and Browserify just by pointing it to the compiler executable.

 
 

How Does It Work?

Amok works by connecting directly to the browser via the remote debugging protocol, which means no browser extension are required.

Using a browser debugging session like this, means that it is truly a live editing experience with support from the JavaScript runtime itself, so the application state is never lost and it can reach deep down into closures without causing side effects.

Where Can I Get It?

Download and install node

Open a terminal window and run the following command

npm install --global amok
 
 

How Do I Use It?

First, open Google Chrome with remote debugging enabled

Open a terminal window, navigate to your project and run the command with the files you want live edit

amok app.js
